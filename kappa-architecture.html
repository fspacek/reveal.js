Architecture: document based access

  • current day computed on the fly

  • SQL reads several rows + map / reduce

Experiences

Architecture: Kappa Architecture

  • Mongo DB
  • flat files = journal
  • journal ZIPped and offloaded
  • indexes for day / month computed on the fly (EhCache evict)
  • Kryo serialized binary in Mongo DB binary field
  • querying several documents + map / reduce (RxJava)
  • no nightly jobs

Experiences

  • input stream serialization - OK
  • handles milions of records query in 0,5 secs - OK
